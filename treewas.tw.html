A module for the main logic of the treeWAS algorithm.

treewas.tw.terminal_score(genes, traits, trait_type='discrete', sign=True)[source]

Calculates the treeWAS terminal score (score 1).

The column names of the gene DataFrame need to be the same as the index of the trait DataFrame. The order also needs to match.

Parameters:

  • genes (pandas.DataFrame) – A DataFrame containing binary genetic data. Rows represent loci and columns individuals.

  • traits (pandas.DataFrame) – A DataFrame containing phenotypic data. Rows represent individuals and columns traits.

  • trait_type (str) – String indicating the type of trait, either "discrete", "continuous" or "categorical"

  • sign (bool) – Flag indicating whether the returned scores should be signed. (default is True)

Returns:

DataFrame with the terminal scores, where rows represent loci and columns represent traits.

Return type:

pandas.DataFrame

Raises:

ValueError – If specified trait_type is not admissible

treewas.tw.simulate_loci(n_sim, dist, tree, node_names, seed=None)[source]

Performs the simulation of genetic loci under the assumption of no association between genotype and phenotype.

Parameters:

  • n_sim (int) – Number of loci to be simulated. Should be at least then times the number of empirical loci.

  • dist (numpy.ndarray) – Homoplasy distribution for sampling per locus number of substitutions.

  • tree (TreeWrapper) – Reconstructed phylogenetic tree.

  • node_names (list[str]) – A list with the names of all the nodes in the tree.

  • seed (int) – Seed for pseudo-random number generator. (default is None)

Returns:

A DataFrame containing the simulated loci. Rows represent loci and columns nodes in the tree.

treewas.tw.treewas(genes, traits, tree, trait_type, n_sim=None, homoplasy_distribution=None, test=('terminal', 'simultaneous', 'subsequent'), gene_reconstruction=None, sim_gene_reconstruction='parsimony', trait_reconstruction=None, p_value_by='count', seed=None)[source]

Runs the treeWAS algorithm

Parameters:

  • genes (pandas.DataFrame) – A DataFrame containing binary genetic data. Columns represent individuals and rows represent loci. Can also contain reconstructed genotypes at internal nodes of the tree. If no reconstruction is provided gene_reconstruction has to be specified.

  • traits (pandas.DataFrame) – A DataFrame containing trait data, Columns represent traits and rows represent isolates. Can also contain reconstructed phenotypes. If no reconstruction is provided, trait_reconstruction has to be specified.

  • tree (TreeWrapper) – The reconstructed phylogeny of the samples. All nodes have to be named and the names should correspond to the names of the samples.

  • trait_type (str) – A string indicating the type of trait, one of discrete, continuous, or categorical.

  • n_sim (int) – An integer indicating the number of genetic loci to be simulated.

  • homoplasy_distribution (numpy.ndarray) – An array containing distribution of the minimum number of substitutions required to explain the observed genotype

  • test (tuple[str, ...]) – A tuple containing the names of the tests to be performed.

  • gene_reconstruction (str) – A string indicating the type of reconstruction to be performed on the genetic data. Either parsimony or ML.

  • sim_gene_reconstruction (str) – A string indicating the type of reconstruction to be performed on the simulated genotype. Either parsimony or ML.

  • trait_reconstruction (str) – A string indicating the type of reconstruction to be performed on the phenotype. Either parsimony or ML.

  • p_value_by (str) – A string indicating the method to calculate the p-values. Either count or density.

  • seed (int) – The seed for the simulation.

Returns:

A dictionary containing pandas DataFrames with the uncorrected p-values. The keys are the values provided in test

Return type:

dict[str, pandas.DataFrame]

treewas.tw.run_treewas(gene_path, trait_path, tree_path, out_dir, trait_type, delimiter=',', homoplasy_path=None, n_sim=None, test=('terminal', 'simultaneous', 'subsequent'), gene_reconstruction=None, sim_gene_reconstruction='parsimony', trait_reconstruction=None, filter_na=True, base_p=0.05, p_value_correction=None, p_value_by='count', seed=None)[source]

Wrapper function for treewas() for command line compatibility

Gene and trait input files are expected to be csv like. The delimiter can be specified in delimiter.

Parameters:

  • gene_path (str) – The path to the file containing the gene data. In the file, columns should represent individuals and rows should represent loci. Can also contain reconstructed genotypes at internal nodes of the tree. If no reconstruction is provided gene_reconstruction has to be specified.

  • trait_path (str) – The path to the file containing the trait data. Columns should represent traits and rows should represent isolates. Can also contain reconstructed phenotypes. If no reconstruction is provided, trait_reconstruction has to be specified.

  • tree_path (str) – The path to the Newick file with the reconstructed phylogeny of the sample. All nodes have to be named and the names should correspond to the names of the samples.

  • out_dir (str) – Path to the directory where the ouput should be written.

  • trait_type (str) – A string indicating the type of trait, one of discrete, continuous, or categorical.

  • delimiter (str) – The delimiter that is used in the input files. Defaults to ",".

  • homoplasy_path (str) – The path to the text file containing the homoplasy distribution.

  • n_sim (int) – An integer indicating the number of genetic loci to be simulated.

  • test (tuple[str, ...]) – A tuple containing the names of the tests to be performed.

  • gene_reconstruction (str) – A string indicating the type of reconstruction to be performed on the genetic data. Either parsimony or ML.

  • sim_gene_reconstruction (str) – A string indicating the type of reconstruction to be performed on the simulated genotype. Either parsimony or ML.

  • trait_reconstruction (str) – A string indicating the type of reconstruction to be performed on the phenotype. Either parsimony or ML.

  • filter_na (bool) – A boolean indicating whether to filter out loci that contain more than 75% missing values.

  • base_p (float) – The base p-value cutoff for determining significant loci.

  • p_value_correction – A string indicating which method to use for p-value correction.

  • p_value_by (str) – A string indicating the method to calculate the p-values. Either count or density.

  • seed (int) – The seed for the simulation.

Returns:

A dictionary containing pandas DataFrames with the uncorrected p-values. The keys are the values provided in test.

Return type:

dict[str, pandas.DataFrame]
